\section*{仓库与位置}

上游组位置位于\href{http://10.42.0.252:12123/general/sdk/rockchip}{General/SDK/rockchip}，几个子仓库分别内容如下：

\begin{enumerate}
    \item \text{RK3588\_Buildroot}对应\code{/buildroot}
    \item \text{RK3588\_Device}对应\code{/device}
    \item \text{RK3588\_ImageParts}对应\code{/image\_parts}
    \item \text{RK3588\_Kernel}对应\code{/kernel}
    \item \text{RK3588\_rkbin}对应\code{/rkbin}
\end{enumerate}

\noindent 其中\text{RK3588\_ImageParts}打包了包括如下的分区：

\begin{enumerate}
    \item \code{app}，应用程序
    \item \code{oemven}，可替换\text{OEM}资源
    \item \code{vendor}，制造商分区，用于保留\text{lib}库等
    \item \code{hold}，不更变分区，用于保留永久数据等
    \item \code{userdata}，用户数据分区
\end{enumerate}

\section*{文档结构}

第一章将讲述各类系统模块如何调用，适用于项目系统发布人员、应用开发人员。

第三章将讲述\text{EMMC}分区设计，适用于系统发布人员。

\subsection{GPIO}

\text{HGD}将所有需要被调用的\text{GPIO}管脚均封装为了\code{/dev/jp\_hgd\_gpio\_ctl\_enable\_\*}设备，列表如下：

\begin{lstlisting}[language=sh]
/dev/jp_hgd_gpio_ctl_enable_evf
/dev/jp_hgd_gpio_ctl_enable_panel
/dev/jp_hgd_gpio_ctl_enable_laser
/dev/jp_hgd_gpio_ctl_enable_refrigerator
\end{lstlisting}

对于上述“设备”，均可以使用\code{echo 1}或者\code{echo on}来进行使能，例如：

\begin{lstlisting}[language=sh]
echo 1 > /dev/jp_hgd_gpio_ctl_enable_evf
\end{lstlisting}

\subsection{UART}

当前的系统中主要提供如下的串口：

\begin{enumerate}
    \item \code{/dev/ttyS1}，电源\text{MCU}
    \item \code{/dev/ttyS2}，调试串口
    \item \code{/dev/ttyS4}，对焦\text{MCU}
    \item \code{/dev/ttyS7}，连接\text{GPS}
    \item \code{/dev/ttyS9}，连接红外机芯
\end{enumerate}

系统提供了一个\code{/root/app/utils/uart_listen}的串口监听程序，该程序可以配置串口波特率，并进行监听。以对焦\text{MCU}通讯为例：

\begin{lstlisting}[language=sh]
# Run uart_listen
./uart_listen 115200 /dev/ttyS4 &
# Send message to UART4
echo -en '\x43\x44\x4A\x50\x02\x02\x01\x00\x10\x0C' > /dev/ttyS4
# output of uart_listen
Received data: 43 44 4A 50 02 02 01 00 10 FE F2 
\end{lstlisting}

\subsection{按键}

系统一共提供了三组按键，其中两组为\text{ADC Key}、一组为\text{GPIO Key}，以\code{evtest}为例：

\begin{lstlisting}[language=sh]
# evtest 
No device specified, trying to scan all of /dev/input/event*
Available devices:
/dev/input/event0:	rk805 pwrkey
/dev/input/event1:	goodix-ts
/dev/input/event2:	adc-keys-ch1
/dev/input/event3:	adc-keys-ch3
/dev/input/event4:	rockchip-hdmi0 rockchip-hdmi0
/dev/input/event5:	gpio-keys
\end{lstlisting}

\noindent 其中，按键定义如下：

\begin{enumerate}
    \item \code{/dev/input/event2} \text{adc-keys-ch1}
        \begin{itemize}
            \item Event type 0 (EV\_SYN)
            \item Event type 1 (EV\_KEY)
                \begin{enumerate}[label=\arabic*.]
                    \item Event code 2 (KEY\_1)
                    \item Event code 3 (KEY\_2)
                    \item Event code 4 (KEY\_3)
                    \item Event code 5 (KEY\_4)
                    \item Event code 6 (KEY\_5)
                \end{enumerate}
        \end{itemize}

    \item \code{/dev/input/event3} \text{adc-keys-ch3}
        \begin{itemize}
            \item Event type 0 (EV\_SYN)
            \item Event type 1 (EV\_KEY)
                \begin{enumerate}[label=\arabic*.]
                    \item Event code 59 (KEY\_F1)
                    \item Event code 60 (KEY\_F2)
                    \item Event code 61 (KEY\_F3)
                    \item Event code 62 (KEY\_F4)
                    \item Event code 63 (KEY\_F5)
                    \item Event code 64 (KEY\_F6)
                \end{enumerate}
        \end{itemize}

    \item \code{/dev/input/event5} \text{gpio-keys}
        \begin{itemize}
            \item Event type 0 (EV\_SYN)
            \item Event type 1 (EV\_KEY)
                \begin{enumerate}[label=\arabic*.]
                    \item Event code 30 (KEY\_A)
                    \item Event code 31 (KEY\_S)
                    \item Event code 46 (KEY\_C)
                \end{enumerate}
        \end{itemize}
\end{enumerate}

\subsection{USB}

系统在\code{/root/sys/usb}下供\text{USB}使能的脚本：

\begin{lstlisting}[language=sh]
adb.sh  mtp.sh    
\end{lstlisting}

\noindent 其中，\code{adb.sh}开启\text{ADB}和\text{RNDIS}功能；\code{mtp}开启\text{MTP}服务。\text{MTP}服务由\code{mpt-server}实现，如有自定义需求，可以自行修改\text{SDK-Buildroot}中的相关配置。

\subsection{待机与唤醒}

\text{HGD}目前支持以如此的方式进行休眠：

\begin{lstlisting}[language=sh]
echo freeze > /sys/power/state
\end{lstlisting}

在休眠后，可以通过按下右侧手柄上方的物理按键\code{C}，将其恢复至休眠前状态。

\textbf{注意}：休眠状态下不会单独关闭制冷机、\text{EVF}等外设的电源\text{GPIO}，此部分应该由用户通过\text{UI}进行手动关闭或暂停。

\subsection{屏幕}

\text{HGD}包含三个屏幕：

\begin{enumerate}
    \item \text{5.5}寸主屏幕，\code{/sys/class/drm/card0-DSI-1}
    \item \text{0.49}寸电子取景器，\code{/sys/class/drm/card0-DSI-2}
    \item \text{HDMI}输出接口，\code{/sys/class/drm/card0-HDMI-A-1}
\end{enumerate}

\subsubsection{MIPI-DSI}

\text{MIPI-DSI}屏幕控制一般包含\text{GPIO}电源使能和\text{MIPI-DSI}输出控制两个部分。

以电子取景器的上电并输出信号为例：

\begin{lstlisting}[language=sh]
# Power on evf
echo 1 > /dev/jp_hgd_gpio_ctl_enable_evf
# Enable DSI's output
echo on > /sys/class/drm/card0-DSI-2/status
\end{lstlisting}

以电子取景器的关闭输出并下电为例：

\begin{lstlisting}[language=sh]
# Disable DSI's output which shown as pure withe now
echo off > /sys/class/drm/card0-DSI-2/status
# Power off evf
echo 0 > /dev/jp_hgd_gpio_ctl_enable_evf
\end{lstlisting}

\section{FS-Overlay}

\subsection{启动脚本}

系统在加载文件系统后，通过\code{/etc/init.d/rcS}，按照字典顺序调用其目录下的\text{S*}脚本进行初始化工作。其中大部分脚本为原版\text{SDK}自带，在\text{FS-Overlay}中新增了如下的脚本。

\begin{enumerate}
    \item \code{S01automount}，挂载\text{MMC}分区；
    \item \code{S02bootanimation}，启用应用层的开机动画；
    \item \code{S49weston}，无功能，仅屏蔽\text{SDK}原有的\text{weston}启动流程；
    \item \code{S50launcher}，无功能，仅屏蔽\text{SDK}原有的\text{QT}桌面启动流程；
    \item \code{S50wifi}，加载\text{WiFi}驱动；
    \item \code{S60evf}，初始化电子取景器功能，然后关闭；
    \item \code{S80launcher}，无功能，仅屏蔽\text{SDK}原有的\text{QT}桌面启动流程；
    \item \code{S98bootanimation_stop}，结束应用层开机动画；
    \item \code{S99vendor}，开启应用层依赖程序，并跳转到\code{/app}下的开机脚本。
\end{enumerate}
